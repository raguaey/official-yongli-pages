# official-yongli-pages

本仓库用于归档和发布多个独立 HTML 页面。所收录页面均为静态内容，不针对任何特定网站或域名。

## 项目简介

本项目是一个通用 HTML 页面归档仓库。页面内容以独立、自包含的形式存放，可通过 GitHub Pages 直接访问。仓库本身不提供搜索、索引或导航功能，仅作为页面存储与版本管理的载体。

## 目录说明

```
├── index.html        # 仓库首页（可选）
├── pages/            # 存放所有归档页面的子目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── README.md         # 本文件
```

- `pages/` 目录为页面存放位置，可根据需要自行组织子目录。
- 每个 HTML 文件应保持独立，无外部依赖（除标准 CDN 或公共库外）。

## 页面归档说明

- 所有页面均为静态 HTML，不含后端逻辑。
- 页面资源（CSS、JS、图片等）建议内联或使用可靠公共 CDN。
- 本仓库不保证页面内容的长期可用性或兼容性。
- 页面更新通过常规 Git 提交进行，历史版本可追溯。

## 维护说明

- 欢迎提交 Pull Request 添加或更新页面。
- 请确保新增页面不包含恶意代码、跟踪脚本或违反平台政策的内容。
- 建议在提交前对页面进行基本的链接与样式检查。
- 仓库维护者保留审核与合并的权利。

## 使用方式

1. 克隆或 Fork 本仓库。
2. 在 `pages/` 目录下添加 HTML 文件。
3. 通过 GitHub Pages 设置发布（通常为 `main` 分支的根目录或 `/docs` 目录）。
4. 访问 `https://<你的用户名>.github.io/official-yongli-pages/pages/<文件名>.html` 查看页面。
